The onFallBack callback is fired when a retry-able exception occurred either because internet is not available and the model files were not downloaded for the first time as well. Use onFallBack to handle network access errors.

In no network access scenarios if older ML models are already downloaded, the SDK won't be able to download the new versions and use the pre-downloaded models for predicting responses.
