fun predictAsync(modelId: String, inputs: Array<Any>, callback: PredictionLoadCallback)
Content copied to clipboard

Runs the Inference pipeline on the device. This is an asynchronous predict call and runs the computation on the coroutine thread specified in NimbleNetOptions. The predict queues the operations if there are multiple predict request for the same model. To optimize resources on the device a single model generates only a single prediction at a time.

The method fails gracefully if initialization is not done or not completed. The onFailure callback is called when initialization failed or invalid inputs. The onFallBack callback is fired when a retry-able exception occurred either because internet is not available and the model files were not downloaded for the first time as well. Use onFallBack to handle network access errors.

NimbleNet.predictAsync(
  "rec_v",
  arrayOf(arrayOf(1,2,3,4,5)),
  object : PredictionLoadCallback {
      override fun onSuccess(output: ModelOutput) {
          Log.d(NE_LOG_TAG, "predictAsync:onSuccess=>" + Thread.currentThread().name)
          val text = output["probabilities"]?.data?.joinToString(",") {
              String.format("%.2f",it)
          }
          val text2 = output["label"]?.data?.joinToString(",") {
              String.format("%d",it)
          }
      }

      override fun onFailure(failure: NimbleNetResult.Failure) {
          Log.d(NE_LOG_TAG, "predictAsync:onFailure" + Thread.currentThread().name)
      }

      override fun onFallBack(fallBack: NimbleNetResult.FallBack) {
          Log.d(NE_LOG_TAG, "predictAsync:onFailure" + Thread.currentThread().name)
      }
  }
)
