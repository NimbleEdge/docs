Only a single worker must be instantiated across an app lifecycle. The initialize method ensures creation of the singleton object if needed or a NoOp if the worker is already created. This method is synchronous and thread safe so getInstance calls across threads do not suffer.

This method is recommended to be called during the onCreate method of application class so that NimbleNet object is alive across the session

The initialize is a synchronous call and it will either throw an error or return successfully. This might increase your app load times since the model loading is happening synchronously but this also guarantees that all the future predict calls will run successfully once the model pipeline is downloaded.

The downloads only happen once and SDK re-downloads if and only if a new pipeline version has been uploaded.If the pipelines are not yet downloaded the initialize queues a download request in parallel and returns. For the first time, your successive calls predict/train will be queued until the download finishes.

class EdgeApplication : Application(){
   override fun onCreate(){
       NimbleNet.initAsync(this, config)
   }
}
