NimbleNet

object NimbleNet
Content copied to clipboard

The NimbleNet object is a global accessor for all the training and inference calls. It stays alive for the entire duration of the application session and monitors device health, job scheduling among other tasks. The SDK manages the complete lifecycle of the NimbleNet object to ensure its availability across the session duration. Even though the NimbleNet object remains in memory for the entire duration, it needs to be activated with a initialize method. All the predict methods can only run successfully once initialization of the NimbleNet object is completed. Before then Predict methods will throw a suppressible error of uninitialized NimbleNet Error.

  • Use initialize if you want to synchronously call the initialize method and wait until it returns or throws an error. initialize might increase your app startup time since it will be loading the model pipelines in memory for future predict calls.

  • Use initAsync is the recommended approach to use parallelism. It initializes the model pipelines and makes them ready for serving inference or training requests in the future in an asynchronous manner.

All the initialization methods should be called in the beginning of the Application Init within the onCreate methods. For Example

class EdgeApplication : Application(){
   override fun onCreate(){
       NimbleNet.initialize(...)
       NimbleNet.initAsync(...)
   }
}

Runs the Inference pipeline on the device. This is a synchronous predict call and runs the computation on the same primary thread from which it was called. The predict queues the operations if there are multiple predict request for the same model. To optimize resources on the device a single model generates only a single prediction at a time. However multiple models can run in parallel.If you want to call asynchronously see predictAsync.
